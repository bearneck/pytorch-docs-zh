# Out 参数说明 {#out-notes}

当用户将一个或多个张量传递给 out= 参数时，约定如下：

- 如果 out 张量没有元素，它将被调整为计算输出的形状、步幅和内存格式。
- 如果 out 张量的形状与计算结果不同，将抛出错误 **或** 该 out 张量将被调整为与输出计算相同的形状、步幅和内存格式，就像没有元素的张量一样。（此调整行为已被弃用，PyTorch 正在更新其运算符以始终抛出错误。）
- 传递具有正确形状的 out= 张量在数值上等同于执行操作并将其结果"安全复制"到（可能已调整大小的）out 张量。在这种情况下，步幅和内存格式将被保留。
- 不支持传递需要梯度的 out= 张量。
- 如果多个张量被传递给 out=，则上述行为独立应用于每个张量。

"安全复制"与 PyTorch 的常规复制不同。对于不参与类型提升的操作，源张量和目标张量的设备和数据类型必须匹配。对于参与类型提升的操作，复制可以到不同的数据类型，但复制的目标不能是比源更低的"类型种类"。PyTorch 有四种类型种类：布尔型、整数型、浮点型和复数型，按此顺序。因此，例如，像 add（参与类型提升）这样的操作，如果给定浮点输入但传递整数 out= 张量，将抛出运行时错误。

请注意，虽然对于形状正确的张量，out= 的数值行为是执行操作然后将其结果"安全复制"，但在幕后，操作可能会重用 out= 张量的存储并融合复制以提高效率。许多操作，如 add，会执行这些优化。此外，虽然 PyTorch 的"out= 约定"如上所述，但 PyTorch 中的许多操作并未正确实现该约定，需要更新。
